Standalone installers: grab one

  • Windows - syndie-1.004a.bin.win32.exe (GPG signature) - 5.8MB
    Download and run the installer. Launch Syndie with the desktop or start menu icons, or with syndie.exe
  • Linux - syndie-1.004a.bin.linux32.exe (GPG signature) - 5.0MB
    Download and run the installer via "java -jar syndie-1.004a.bin.linux32.exe" (yes, really). Launch Syndie with the desktop or KDE menu icons, or with "java -jar bin/syndie.exe".
  • OSX - syndie-1.004a.bin.osx.exe (GPG signature) - 5.5MB
    Download and run the installer via "java -jar syndie-1.004a.bin.osx.exe" (yes, really). Launch Syndie with the icon, or with "java -jar bin/syndie.exe".
  • Any platform (new install) - syndie-1.004a.bin.nodeps.exe (GPG signature) - 2.8MB
    does not include SWT or HSQLDB, so only use this if you can arrange to have swt.jar and hsqldb.jar in your Syndie lib/ directory.
  • Any platform (upgrade) - syndie-1.004a.bin.jar-only.zip (GPG signature) - 2.1MB
    Extracts ./lib/syndie.jar to replace an earlier syndie.jar. This should be used for upgrades.
  • Text only - syndie-1.004a.bin.zip (GPG signature) - 2.8MB
    This unzips into ./syndie-1.004a and does not create any desktop or start menu shortcuts, but can be used on computers without a graphical interface.
    To run Syndie without a GUI, launch it as above, but with the parameter --cli.
  • Prerequisites: jre1.4+, GCJ/GIJ 4.0+, or Kaffe 1.1.7+ (other JVMs may suffice)

Source:

  • syndie-1.004a.src.nodeps.tar.bz2 (GPG signature) - 0.9MB

Unreleased development source:

  • A public monotone repository of the syndie code is available on the netsync server mtn.i2p.net in the syndie branch.
    Not familiar with monotone? See this howto
  • Developers are granted write permission to that repository, and everyone is granted read permission.
  • The list of authorized committers can be found in the mtn-committers file, which includes their monotone public keys, ready to be imported with cat mtn-committers | monotone read. That file can be verified against the above GPG public key with gpg --verify mtn-committers.sig.

To upgrade or reinstall, simply install Syndie again on top of itself. Upgrading or reinstalling does not affect your content or keys, just the software. To completely wipe any old data, identities, or keys, delete the $HOME/.syndie directory.

To uninstall, if you used the installer and created desktop icons or start menu shortcuts, simply launch the included uninstaller. Otherwise, just remove the directory you installed Syndie into (e.g. $HOME/syndie or C:\Documents and Settings\User\syndie).

The Syndie content is stored in $HOME/.syndie or %HOME%\.syndie by default, so you should delete that directory as well if you want to remove the content (and private keys!). To use another location (or to run multiple instances at once), simply pass another root directory on the command line (e.g. syndie.exe C:\nym1)
