Standalone installers:

The standalone installers are currently in a broken state, while the build process is being renovated.

Source:

  • Only monotone is working for now.
  • You will need to provide the hsqldb.jar and swt.jar for your platform (place them in the lib/ directory). Be sure to read the INSTALL for instructions.
  • Prerequisites: ant 1.6.5, jdk1.4+/ GCJ/GIJ 4.0+/ or Kaffe 1.1.7+ (other JDKs may suffice), SWT 3.2+ (3.3M4 or newer includes useful features), HSQLDB 1.8.0.7+ (older releases may work)

Older releases are archived

Development source:

  • A public monotone repository of the syndie code is available on the netsync server mtn.i2p2.de in the syndie branch.
    Not familiar with monotone? See this howto
  • Developers are granted write permission to that repository, and everyone is granted read permission.
  • The list of authorized committers can be found in the mtn-committers file, which includes their monotone public keys, ready to be imported with cat mtn-committers | monotone read. That file can be verified against the above GPG public key with gpg --verify mtn-committers.sig.

To upgrade or reinstall, simply install Syndie again on top of itself. Upgrading or reinstalling does not affect your content or keys, just the software. To completely wipe any old data, identities, or keys, delete the $HOME/.syndie directory.

To uninstall just remove the directory you installed Syndie into (e.g. $HOME/syndie-build or C:\Documents and Settings\User\syndie).

The Syndie content is stored in $HOME/.syndie or %HOME%\.syndie by default, so you should delete that directory as well if you want to remove the content (and private keys!). To use another location (or to run multiple instances at once), simply pass another root directory on the command line (e.g. java -jar syndie.jar %HOME%\nym1)
